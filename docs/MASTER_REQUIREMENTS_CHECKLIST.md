# Master Requirements Checklist

- Foundation: TypeScript, responsive shell, shared domain models, local data, reset, error-safe seed recovery, lint/format/test/build scripts.
- Local data: typed asynchronous repositories for course, catalog, menu, inventory, golf sessions, orders, fulfilments, staff, kitchen, restaurant, members/minimum spend, notifications, and settings.
- Roles: guest, registered golfer, club member, beverage-cart staff, runner, kitchen employee, restaurant server, course manager, platform administrator.
- Course: FairwayServe Demo Club, 18 holes, facilities, delivery zones, alcohol restrictions, meeting points, active round screen.
- Commerce: 40+ products, categories, search-ready metadata, favourites, modifiers, add-ons, fulfilment sources, cart and checkout totals.
- Operations: multi-fulfilment orders, status history, assignment explanations, staff app, inventory, kitchen display, restaurant readiness, simulated payments, minimum spend, alcohol workflow, notifications, management, analytics, demo centre.
- Quality: responsive CSS, focus states, reduced motion, empty/loading states, corrupted-storage recovery, duplicate checkout protection, automated tests.
